# Vehicle Web Viewer — CLAUDE.md

## Project Goal
UE5 없이도 일반 노트북 브라우저에서 차량 3D 모델을 볼 수 있는 웹 뷰어.
GM Technical Center Korea (GMTCK) PQDQ팀 내부용으로 시작, 이후 GMTCK 전체 → GM 글로벌로 확장 예정.

## Background
- 기존: JT → Datasmith → UE5에서만 차량 확인 가능 (고사양 PC 필수)
- 목표: 브라우저만 있으면 누구나 접근 가능
- 서버: Windows 데스크탑 1대 (항상 켜놓음), 추후 Cloud 이전 예정

## Tech Stack
- **Frontend**: React + react-three-fiber (Three.js) + WebGPU Renderer
- **Backend**: FastAPI (Python)
- **3D Format**: glTF 2.0 + Draco 압축
- **VR**: WebXR (Three.js 내장)
- **Pipeline**: Python 스크립트 (FBX → glTF 자동 변환)

## Phase 현황
- [ ] Phase 1: FBX → glTF 변환 파이프라인 + FastAPI 파일 서버
- [ ] Phase 2: 기본 뷰어 (차량 선택 화면 + 회전/줌/이동)
- [ ] Phase 3: 퀄리티 업 (PBR + IBL + 후처리)
- [ ] Phase 4: 인터랙션 (파츠 클릭/숨기기/색상변경/문 애니메이션)
- [ ] Phase 5: WebXR VR 연동 (Meta Quest 우선)
- [ ] Phase 6: 로그인/권한 + Cloud 이전

## 성능 목표 (항상 지킬 것)
- 타겟 기기: 일반 노트북 (Intel 내장 그래픽 포함)
- 초기 로딩 파일 크기: **30MB 이하** (외장 기준)
- 목표 프레임: **30fps 이상**
- 초기 로딩 시간: **5초 이내**
- 동시 접속: 5~20명

## 차량 구조 (glTF 분리 기준)
각 차량은 아래 구역으로 분리해서 별도 glb 파일로 저장:
- `exterior.glb` — 외장
- `interior.glb` — 내장
- `chassis.glb` — 섀시
- `powertrain.glb` — 파워트레인

처음 로딩 시 exterior.glb만 로드, 나머지는 사용자 요청 시 로드 (LOD 전략).

## LOD 규칙 (절대 지킬 것)
- 초기 진입 시 반드시 저해상도(LOD0) 먼저 로드
- 고해상도는 사용자가 구역 선택 시 로드
- 13GB 전체 동시 로딩 절대 금지
- LOD 없이 고해상도 직접 로딩 절대 금지

## 절대 하면 안 되는 것
- `전체 차량 단일 glTF로 로딩` — 용량 초과로 브라우저 크래시
- `WebGL fallback 강제` — WebGPU 기본 사용, WebGL은 fallback으로만
- `서버에서 3D 렌더링 시도` — 렌더링은 항상 클라이언트(브라우저)에서
- `Varjo WebXR 지원` — Varjo는 WebXR 미지원, 이번 범위 밖

## API 설계 원칙
- RESTful FastAPI
- 차량 목록, 구역별 glb 파일 서빙, 메타데이터 제공
- Cloud 이전 고려: 로컬 파일 경로를 환경변수로 관리 (하드코딩 금지)
- CORS 허용 (사내 인트라넷 환경)

## 확장성 고려 (항상 염두에 둘 것)
- 지금은 데스크탑 로컬 서버, 나중에 AWS/Azure로 이전
- 파일 저장 경로는 환경변수로 분리해서 Cloud Storage로 쉽게 교체 가능하게
- 로그인/권한은 Phase 6, 지금은 IP 기반 접근 제어로 충분
