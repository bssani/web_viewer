# Vehicle Web Viewer — CLAUDE.md

## Project Goal
UE5 없이도 일반 노트북 브라우저에서 차량 3D 모델을 볼 수 있는 웹 뷰어.
GM Technical Center Korea (GMTCK) PQDQ팀 내부용으로 시작, 이후 GMTCK 전체 → GM 글로벌로 확장 예정.

## Background
- 기존: JT → Datasmith → UE5에서만 차량 확인 가능 (고사양 PC 필수)
- 목표: 브라우저만 있으면 누구나 접근 가능
- 서버: Windows 데스크탑 1대 (항상 켜놓음), 추후 Cloud 이전 가능성 있음 (확정 아님)

## 확정 기술 스택 (변경 금지)
- **Frontend**: React + Babylon.js
- **렌더러**: WebGPU 우선, 미지원 시 WebGL 2.0 자동 fallback
- **Backend**: FastAPI (Python)
- **3D 포맷**: GLB (텍스처 embedded, 구역별 분리)
- **기하학 압축**: Meshopt (`EXT_meshopt_compression`)
- **텍스처 압축**: KTX2 + Basis Universal (`KHR_texture_basisu`)
- **VR**: WebXR (Babylon.js 내장)
- **파이프라인**: UE5 → 구역별 GLB export → Meshopt/KTX2 압축 → 서버
- **서버 OS**: Windows
- **UE5 기준 버전**: 5.5.2 (GMTCK PQDQ 팀 내 검증 완료 버전, 변경 금지)
  - 변경 필요 시 기존 차량 에셋 전면 재검증 후 본 문서 업데이트

## 성능 목표 (항상 지킬 것)
- 타겟 기기: 일반 노트북 (Intel 내장 그래픽 포함)
- 초기 로딩 파일 크기: 30MB 이하 (exterior.glb 기준)
- 목표 프레임: 노트북 30fps 이상, Meta Quest 3 72fps 이상 (WebXR 진입 시)
- **초기 로딩 시간**:
  - 단일 접속 기준: 5초 이내
  - 동시 4~5명 기준: 5초 이내 (1Gbps 랜 한계 고려)
  - 동시 20명 기준: 최대 15초 (Phase 6 Cloud + CDN 이전 시 해결)
- 동시 접속: 5~20명
- 드로우콜 상한: 구역당 500 이하 (exterior 기준), PBR + IBL + 후처리 적용 후에도 유지
- 머티리얼 수 상한: 구역당 50 이하 — UE5 export 전 머티리얼 병합 필수
- **총 텍스처 메모리 예산**: 구역당 1GB 이내 (Intel 내장 그래픽 기준)
  - 초과 시 Chrome 탭 강제 종료 위험
  - compress.py가 metadata.json에 텍스처 총 바이트 기록 필수

### 네트워크 대역폭 현실 (중요)
Windows 데스크탑 1대 서버 + 1Gbps 사내 랜 기준:
- 이론 최대: 125MB/s, 실제: 80~100MB/s
- 30MB × 5명 동시 요청 = 150MB → 약 1.5~2초
- 30MB × 20명 동시 요청 = 600MB → 약 6~8초

이는 스택 문제가 아니라 **물리적 제약**이며, Phase 6 Cloud 이전(CDN 분산)으로만 근본 해결 가능. 지금은 이 한계를 인지하고 팀 내 사용 패턴(예: 리뷰 미팅 시 순차 접속 권장)으로 관리.

## 렌더러 Fallback 규칙 (필수)
`navigator.gpu` 존재 여부만으로 판정 금지. Intel 내장 그래픽에서 객체는 존재하지만 어댑터 요청이 실패하는 케이스가 빈번함. 반드시 Babylon.js 공식 비동기 체크 사용.

```javascript
// WebGPU 실제 지원 여부 비동기 감지 후 fallback
const webGPUSupported = await BABYLON.WebGPUEngine.IsSupportedAsync;

let engine;
if (webGPUSupported) {
    engine = new BABYLON.WebGPUEngine(canvas);
    await engine.initAsync();
} else {
    engine = new BABYLON.Engine(canvas, true);
}
```

- WebGPU: 최고 퀄리티 (SSR 포함)
- WebGL 2.0: 95% 퀄리티 (SSR 제외), 기능 동일
- 두 렌더러 모두 PBR/IBL/VR/애니메이션 정상 동작

## Meshopt / KTX2 디코더 등록 (필수)
Babylon.js는 `EXT_meshopt_compression`, `KHR_texture_basisu` 확장을 디코더 수동 등록 없이는 로드 실패함. 앱 초기화 시 반드시 등록.

```javascript
// Meshopt 디코더 등록
BABYLON.GLTFValidation.Configuration.meshoptDecoder = MeshoptDecoder;

// KTX2 / Basis 트랜스코더 경로 지정
BABYLON.KhronosTextureContainer2.URLConfig.jsDecoderModule = "/libs/babylon.ktx2Decoder.js";
BABYLON.KhronosTextureContainer2.URLConfig.wasmUASTCToASTC = "/libs/ktx2Transcoders/uastc_astc.wasm";
BABYLON.KhronosTextureContainer2.URLConfig.wasmUASTCToBC7 = "/libs/ktx2Transcoders/uastc_bc7.wasm";
// 나머지 트랜스코더도 동일하게 등록
```

디코더 파일은 `frontend/public/libs/`에 정적으로 배치.

## KTX2 트랜스코딩 포맷 선택 기준 (필수)
Basis Universal은 런타임에 GPU별로 트랜스코딩됨. Intel 내장 그래픽(타겟 기기)은 ASTC 미지원이 잦아 RGBA8로 풀리면 VRAM이 오히려 증가함. 파이프라인에서 텍스처 종류별로 압축 모드를 구분.

| 텍스처 종류 | 압축 모드 | 이유 |
|---|---|---|
| BaseColor (페인트, 내장재) | ETC1S | 작고 빠름, 퀄리티 손실 허용 |
| Normal Map | UASTC | ETC1S는 노멀 왜곡 심함 |
| Metallic/Roughness | UASTC | PBR 품질 유지 필요 |
| Occlusion | UASTC | 디테일 보존 필요 |
| Emissive | ETC1S | 작고 빠름 |

### KTX2 트랜스코딩 실측 (Phase 2 진입 시 필수)
실제 타겟 노트북 2~3대에서 각 텍스처 slot이 어떤 GPU 포맷으로 풀리는지 Babylon.js 콘솔 로그로 확인. 결과를 벤치마크 기록에 남김. RGBA8로 풀리면 압축 이점 상실.

## 메모리 관리 규칙 (필수)
- 차량 전환 시 `scene.dispose()`만 호출, **`engine.dispose()` 호출 금지**
  - 엔진 재생성은 WebGL 컨텍스트 제한(브라우저당 최대 16개) 위반 위험
  - WebGPU는 컨텍스트 생성 비용이 특히 큼
- `engine.dispose()`는 페이지 언로드(`beforeunload`) 시점에만
- 사용하지 않는 텍스처 즉시 GPU 메모리에서 해제
- 미이행 시 브라우저 탭 강제 종료 발생

### 메모리 계측 (dev 모드 필수)
차량 전환 전후에 다음 값을 콘솔 로깅. 누수 감지 없이는 dispose 규칙 준수 확인 불가.

```javascript
console.log({
    meshes: scene.meshes.length,
    textures: engine.getLoadedTexturesCache().length,
    activeMeshes: scene.getActiveMeshes().length,
    totalVertices: scene.getTotalVertices(),
    // Chrome 전용 (VRAM 대체 지표)
    jsHeap: performance.memory?.usedJSHeapSize,
});
```

차량 전환 후 이 값들이 **0 또는 새 차량의 값으로만** 남아야 함. 누적되면 dispose 실패.

## 캐시 무효화 전략 (필수)
Cache-Control `max-age=3600`만으로는 디자인 업데이트 반영 불가. 이전 버전이 최대 1시간 동안 표시될 수 있어 리뷰 맥락에서 치명적.

### 해시 기반 캐시 버스팅
- compress.py가 압축 완료 후 GLB 파일 SHA-256 해시를 계산하여 metadata.json에 `file_hash` 필드로 기록
- 프론트엔드는 먼저 `/vehicles/{id}` 메타데이터를 fetch 후, GLB URL에 해시를 쿼리 파라미터로 부착:
  ```javascript
  const metadata = await fetch(`/vehicles/${id}`).then(r => r.json());
  const glbUrl = `/vehicles/${id}/exterior?v=${metadata.zones.exterior.file_hash.slice(0, 8)}`;
  ```
- `metadata.json` 자체는 **Cache-Control: no-cache** 로 서빙 (항상 최신 확인)
- GLB 파일은 `max-age=3600` 유지 (해시가 바뀌면 다른 URL이므로 자동 무효화)

### Phase 6 전환
Phase 6에서 파일명 자체에 해시 삽입 (`exterior.a1b2c3.glb`) + `immutable` 캐싱으로 전환.

## 차량 구조 (GLB 분리 기준)
UE5에서 구역별로 직접 분리 export:
- `exterior.glb` — 외장
- `interior.glb` — 내장
- `chassis.glb` — 섀시
- `powertrain.glb` — 파워트레인

처음 로딩 시 exterior.glb만 로드, 나머지는 사용자 요청 시 로드.

## Windows 운영 제약 (필수)

### MAX_PATH 260자 제한
- Windows 기본값은 경로 전체 260자 제한
- UE5 export 폴더 + 프로젝트 폴더 깊이를 더하면 쉽게 초과
- **회피 전략**:
  1. Windows 10 1607+ 레지스트리에서 긴 경로 활성화 (`HKLM\SYSTEM\CurrentControlSet\Control\FileSystem\LongPathsEnabled=1`)
  2. Python 코드에서 장 경로 필요 시 `\\?\` 프리픽스 사용 (`\\?\C:\very\long\path`)
  3. 프로젝트 루트는 가능한 얕은 경로에 설치 (예: `C:\gmtck\vwv\`)
- 설치 가이드에 "프로젝트 루트 경로 깊이 최대 80자 권장" 명시

### 파일명 소문자 강제
- Windows는 대소문자 무시하지만 HTTP URL과 Linux는 구분
- compress.py가 저장하는 모든 파일명은 **소문자로 강제 변환**
- `Exterior.glb` → `exterior.glb`
- Cloud 이전(Linux) 시 대소문자 불일치로 깨지는 문제 사전 차단

### 네트워크 드라이브 접근
- 매핑된 드라이브 문자(`Z:\`) 의존 금지 — 서비스 실행 시 매핑 없음
- UNC 경로(`\\server\share`) 직접 사용
- 서비스 실행 계정에 해당 UNC 경로 접근 권한 필요

### 서비스 등록 (권장)
- `uvicorn`, `watch.py` 모두 Windows 재부팅 시 자동 재시작 필요
- NSSM (Non-Sucking Service Manager) 사용 권장
- Phase 1 배포 시 서비스 등록 가이드 작성

### 전원 관리
- Windows 기본 전원 설정이 절전 모드로 전환되면 서버 중단
- "고성능" 또는 "항상 켜짐"으로 설정 변경 필수

### 방화벽
- Windows Defender 방화벽 인바운드 규칙: 포트 8000 (FastAPI) 허용

## API 설계
| Method | Path | 설명 | Cache-Control |
|--------|------|------|---|
| GET | `/vehicles` | 전체 차량 목록 | no-cache |
| GET | `/vehicles/{id}` | 차량 메타데이터 | no-cache |
| GET | `/vehicles/{id}/{zone}` | 구역별 GLB 파일 서빙 | public, max-age=3600 |

### 서빙 규칙
- GLB는 `FileResponse` 대신 `StaticFiles` 마운트로 서빙 — 동시 접속 시 이벤트 루프 블로킹 방지
- GLB Cache-Control: `public, max-age=3600` (해시 기반 캐시 버스팅과 병행)
- metadata.json Cache-Control: `no-cache` (항상 최신 확인)
- **CORS는 사내 도메인 화이트리스트만 허용** (전체 허용 금지)
  ```python
  allow_origins=[
      "http://<사내-서버-IP>",
      "http://<사내-도메인>",
  ]
  ```
- 인증 미들웨어 skeleton 배치 (Phase 1~5: no-op, Phase 6: JWT 검증 추가)

### metadata.json 스키마
compress.py가 자동 생성. 뷰어가 로딩 전략 수립 및 캐시 무효화에 사용.

```json
{
  "vehicle_id": "string",
  "vehicle_name": "string",
  "created_at": "ISO8601",
  "updated_at": "ISO8601",
  "ue_version": "5.5.2",
  "zones": {
    "exterior": {
      "file": "exterior.glb",
      "file_size_bytes": 0,
      "file_hash": "sha256-hex",
      "draw_calls": 0,
      "material_count": 0,
      "vertex_count": 0,
      "texture_memory_bytes": 0
    }
  }
}
```

- `created_at`: 최초 생성 시에만 기록
- `updated_at`: 매번 갱신
- `file_hash`: SHA-256, 캐시 무효화 키
- `texture_memory_bytes`: 모든 텍스처를 디코딩했을 때 예상 메모리 (텍스처 메모리 예산 감시용)

## 파이프라인 스크립트 책임 분리
- `watch.py`: UE5 export 폴더 감시, 파일 안정화 판정, `compress.py` 호출
- `compress.py`: Meshopt + KTX2 압축, metadata.json 생성, 결과를 `models_dir`로 이동
- 두 스크립트는 독립 실행 가능해야 함 (compress.py 단독 호출로 배치 처리 가능)
- `compress.py`는 **CLI와 Python 모듈 양쪽으로 호출 가능**해야 함:
  - `def compress_glb(input_path, vehicle_id, zone, ...) -> dict` 함수로 핵심 로직 분리
  - CLI는 함수를 호출하는 얇은 래퍼
  - Phase 6에서 HTTP 업로드 엔드포인트가 같은 함수 재사용

### 파일 안정화 판정 규칙 (watch.py 필수)
UE5 export는 청크 단위로 쓰기 때문에 일시적 쓰기 중단을 완료로 오판 금지. 다음 **모두 충족**해야 압축 시작:

1. 파일 크기가 **5초 이상** 변화 없음
2. mtime이 **5초 이상** 변화 없음
3. Windows 파일 락 해제 확인 (**`rb+` 쓰기 모드**로 open 시도 성공, `rb`는 안됨)

셋 중 하나라도 실패 시 대기 지속.

### 중복 실행 방지 (락 파일)
디바운스만으로는 부족. 여러 watch.py 인스턴스, 수동 CLI, 이전 프로세스 잔존 등 다양한 케이스 대비:
- compress.py 시작 시 `{vehicle_id}_{zone}.lock` 파일 생성
- 락 파일에 PID + 생성 시각 기록
- 다른 프로세스는 락 파일 존재 시 대기 또는 스킵
- 프로세스 크래시 대비: 락 파일의 PID가 살아있지 않거나 생성 10분 경과 시 stale로 판정하여 무시

### 압축 효율 역전 방지
이미 압축된 GLB를 재압축하거나 텍스처가 작을 때 결과물이 더 커질 수 있음.
- 압축 결과 크기가 원본보다 크면 원본 유지
- 로그에 기록 후 해당 zone은 원본 파일로 metadata 생성

## 확장성 추상화 (Phase 1부터 적용)

- Phase 1~5: 콘솔 + 파일 로그 (텍스트)
