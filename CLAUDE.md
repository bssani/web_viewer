# Vehicle Web Viewer — CLAUDE.md

## Project Goal
UE5 없이도 일반 노트북 브라우저에서 차량 3D 모델을 볼 수 있는 웹 뷰어.
GM Technical Center Korea (GMTCK) PQDQ팀 내부용으로 시작, 이후 GMTCK 내 여러 팀(디자인, 엔지니어링 등)으로 확장.

## Background
- 기존: JT → Datasmith → UE5에서만 차량 확인 가능 (고사양 PC 필수)
- 목표: 브라우저만 있으면 누구나 접근 가능 (앱 설치 없음)
- 서버: Windows 데스크탑 1대 (항상 켜놓음), 추후 Cloud 이전 가능성 있음 (확정 아님)
- **v1.0 = Phase 4** 완료 시점 (인증 제외, 사내 인트라넷 배포)

### 사용 시나리오
- **시나리오 A (리뷰 미팅)**: 회의실 대형 모니터에 띄우고, 디자이너/엔지니어가 함께 차량 확인. 회전/확대, 색상 변경, 파츠 숨기기로 비교 검토.
- **시나리오 B (개인 검토)**: 각자 자기 노트북에서 차량 확인. 빠른 확인 위주.
- 협업 기능(주석, 실시간 동기화)은 v1.0 이후 검토.

### 사용자 범위
- Phase 2~4: GMTCK 내 여러 팀 (PQDQ, 디자인 센터, 파츠 부서 등)
- Phase 6+: GM 글로벌 확장 가능성 (확정 아님)

## 확정 기술 스택 (변경 금지)

### Backend
- **서버**: FastAPI (Python)
- **서버 OS**: Windows
- **3D 포맷**: GLB (텍스처 embedded, 구역별 분리)
- **기하학 압축**: Meshopt (`EXT_meshopt_compression`)
- **텍스처 압축**: KTX2 + Basis Universal (`KHR_texture_basisu`)
- **파이프라인**: UE5 → 구역별 GLB export → Meshopt/KTX2 압축 → 서버
- **UE5 기준 버전**: 5.5.2 (GMTCK PQDQ 팀 내 검증 완료 버전, 변경 금지)

### Frontend
- **프레임워크**: React (Vite + TypeScript)
- **3D 엔진**: Babylon.js
- **렌더러**: WebGPU 우선, 미지원 시 WebGL 2.0 자동 fallback
- **UI 스타일링**: Tailwind CSS
- **상태 관리**: useState / useContext (Phase 4에서 Zustand 도입 검토)
- **Babylon.js 통합**: 순수 useEffect (react-babylonjs 라이브러리 사용 금지)
- **VR**: WebXR (선택적, Phase 5 — 하단 참조)

## 성능 목표 (항상 지킬 것)
- 타겟 기기: 일반 노트북 (Intel 내장 그래픽 포함)
- 초기 로딩 파일 크기: 30MB 이하 (exterior.glb 기준)
- 목표 프레임: 노트북 30fps 이상
- **초기 로딩 시간**:
  - 단일 접속 기준: 5초 이내
  - 동시 4~5명 기준: 5초 이내 (1Gbps 랜 한계 고려)
  - 동시 20명 기준: 최대 15초 (Phase 6 Cloud + CDN 이전 시 해결)
- 동시 접속: 5~20명
- 드로우콜 상한: 구역당 500 이하 (exterior 기준), PBR + IBL + 후처리 적용 후에도 유지
- 머티리얼 수 상한: 구역당 50 이하 — UE5 export 전 머티리얼 병합 필수
- **총 텍스처 메모리 예산**: 구역당 1GB 이내 (Intel 내장 그래픽 기준)

### 네트워크 대역폭 현실 (중요)
Windows 데스크탑 1대 서버 + 1Gbps 사내 랜 기준:
- 이론 최대: 125MB/s, 실제: 80~100MB/s
- 30MB × 5명 동시 요청 = 150MB → 약 1.5~2초
- 30MB × 20명 동시 요청 = 600MB → 약 6~8초

Phase 6 Cloud 이전(CDN 분산)으로만 근본 해결 가능. 지금은 팀 내 사용 패턴(리뷰 미팅 시 순차 접속 권장)으로 관리.

## 렌더러 Fallback 규칙 (필수)
`navigator.gpu` 존재 여부만으로 판정 금지. 반드시 Babylon.js 공식 비동기 체크 사용.

```javascript
const webGPUSupported = await BABYLON.WebGPUEngine.IsSupportedAsync;

let engine;
if (webGPUSupported) {
    engine = new BABYLON.WebGPUEngine(canvas);
    await engine.initAsync();
} else {
    engine = new BABYLON.Engine(canvas, true);
}
```

- WebGPU: 최고 퀄리티 (SSR 포함)
- WebGL 2.0: 95% 퀄리티 (SSR 제외), 기능 동일
- **개발자 모드 패널에 현재 사용 중인 엔진 표시 필수**

## Meshopt / KTX2 디코더 등록 (필수)
Babylon.js는 `EXT_meshopt_compression`, `KHR_texture_basisu` 확장을 디코더 수동 등록 없이는 로드 실패함. 앱 초기화 시 반드시 등록.

```javascript
BABYLON.GLTFValidation.Configuration.meshoptDecoder = MeshoptDecoder;
BABYLON.KhronosTextureContainer2.URLConfig.jsDecoderModule = "/libs/babylon.ktx2Decoder.js";
BABYLON.KhronosTextureContainer2.URLConfig.wasmUASTCToASTC = "/libs/ktx2Transcoders/uastc_astc.wasm";
BABYLON.KhronosTextureContainer2.URLConfig.wasmUASTCToBC7 = "/libs/ktx2Transcoders/uastc_bc7.wasm";
```

디코더 파일은 `frontend/public/libs/`에 정적으로 배치.

## KTX2 트랜스코딩 포맷 선택 기준 (필수)
| 텍스처 종류 | 압축 모드 | 이유 |
|---|---|---|
| BaseColor (페인트, 내장재) | ETC1S | 작고 빠름, 퀄리티 손실 허용 |
| Normal Map | UASTC | ETC1S는 노멀 왜곡 심함 |
| Metallic/Roughness | UASTC | PBR 품질 유지 필요 |
| Occlusion | UASTC | 디테일 보존 필요 |
| Emissive | ETC1S | 작고 빠름 |

### KTX2 트랜스코딩 실측 (Phase 2 진입 시 필수)
실제 타겟 노트북 2~3대에서 각 텍스처 slot이 어떤 GPU 포맷으로 풀리는지 확인. 결과를 벤치마크 기록에 남김.

## 메모리 관리 규칙 (필수)
- 차량 전환 시 `scene.dispose()`만 호출, **`engine.dispose()` 호출 금지**
- `engine.dispose()`는 페이지 언로드(`beforeunload`) 시점에만
- 사용하지 않는 텍스처 즉시 GPU 메모리에서 해제

## 캐시 무효화 전략 (필수)
### 해시 기반 캐시 버스팅
- compress.py가 GLB 파일 SHA-256 해시를 metadata.json `file_hash` 필드에 기록
- 프론트엔드는 먼저 `/vehicles/{id}` 메타데이터 fetch 후, GLB URL에 해시 쿼리 파라미터 부착
- metadata.json: `Cache-Control: no-cache`
- GLB: `Cache-Control: public, max-age=3600`

## 차량 구조 (GLB 분리 기준)
- `exterior.glb` — 외장 (초기 로딩 대상)
- `interior.glb` — 내장 (요청 시 로드)
- `chassis.glb` — 섀시 (요청 시 로드)
- `powertrain.glb` — 파워트레인 (요청 시 로드)

## 파이프라인 스크립트 책임 분리
- `watch.py`: UE5 export 폴더 감시, 파일 안정화 판정, `compress.py` 호출
- `compress.py`: Meshopt + KTX2 압축, metadata.json 생성, 결과를 `models_dir`로 이동
- compress.py는 CLI + Python 모듈 양쪽 호출 가능
